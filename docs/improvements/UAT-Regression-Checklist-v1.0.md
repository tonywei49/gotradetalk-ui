## 0. 已完成基线
- [x] `staff` 登录 smoke（Playwright）
- [x] 附件 `上传 -> 发送 -> 撤回` smoke（Playwright）
- [x] 群聊邀请/加入/移除 API 回归（`scripts/regression-matrix.mjs`）

---

## 1. Toast 统一提示系统
### 目标
- 统一成功/失败提示风格，避免“没反应”。

### 自动化
- [ ] 提交后出现成功提示（登录、发送、上传、删除、邀请）
- [ ] 权限不足统一提示（`M_FORBIDDEN`）
- [ ] 配额超限统一提示（`M_LIMIT_EXCEEDED` / 413）
- [ ] 网络超时统一提示（fetch timeout / disconnect）

### 手工
- [ ] 中文/英文文案一致性核对
- [ ] client/staff 两入口提示一致性核对

### 通过标准
- [ ] 高优先失败场景均有可见提示（<2s）
- [ ] 无“按钮点击后无反馈”场景

---

## 2. 文件上传中断与恢复策略
### 目标
- 保证断网、刷新、切房间、重连后草稿状态可预期。

### 自动化
- [ ] 上传中断后显示失败状态且可重试
- [ ] 切换房间后返回，草稿仍可见且状态正确
- [ ] 刷新后草稿恢复策略符合设计（保留或清理）
- [ ] 重连后重试队列可继续执行

### 手工
- [ ] DevTools 手动断网/恢复，验证 UI 状态变化
- [ ] 多文件并发上传时逐个失败/成功可区分

### 通过标准
- [ ] 无“幽灵草稿”或“服务端呆账”不可见文件
- [ ] 重试行为可解释，状态可追踪

---

## 3. 文件中心性能与大数据量
### 目标
- 在大量附件场景下保持可用性与可操作性。

### 自动化
- [ ] 搜索去抖触发次数受控（录 trace）
- [ ] 批量删除时有 loading/禁用态
- [ ] 列表渲染性能基线（首屏渲染时间、交互延迟）

### 手工
- [ ] 1000+ 附件数据下滚动与筛选体验
- [ ] 分页或虚拟列表策略验证
- [ ] 批量操作耗时反馈与错误回传验证

### 通过标准
- [ ] 搜索/筛选无明显卡顿
- [ ] 批量操作期间有清晰进度反馈

---

## 4. 前端交付收尾（部署与 UAT）
### 目标
- 完成正式交付前的部署与验收闭环。

### 自动化
- [ ] Nginx 镜像构建成功
- [ ] 关键 smoke（登录/收发/附件/群聊）在镜像环境通过

### 手工
- [ ] UAT 清单逐项签核（Client/Staff/群聊/翻译/移除）
- [ ] 回滚流程演练（镜像 tag 回退）

### 通过标准
- [ ] 镜像可重复构建与部署
- [ ] UAT 清单全绿且有记录

---

## 5. 多端一致性验证（UI vs Element）
### 目标
- 核对 Matrix 事件层行为一致，避免 UI 偏差。

### 自动化
- [ ] 同房间同操作下，事件类型与 membership 变化一致
- [ ] 邀请/加入/离开/移除流程事件链一致

### 手工
- [ ] 关键流程双端并行操作对照（UI + Element）
- [ ] 时间戳与系统消息内容可对齐

### 通过标准
- [ ] 无“只在自研 UI 出现、Element 不出现”的关键行为偏差

---

## 执行顺序建议
1. Toast 统一（先把失败可见性补齐）
2. 上传中断恢复（先稳）
3. 文件中心性能（再快）
4. Nginx + UAT（交付门槛）
5. UI vs Element 对照（最终一致性验收）
