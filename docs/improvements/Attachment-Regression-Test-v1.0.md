# GoTradeTalk 附件與群聊回歸測試清單 v1.0

> 適用版本：`gotradetalk-ui` `main` 分支（含 commit `29b39c8` 之後）
> 目標：覆蓋群聊邀請/移除/再邀請與附件草稿、送出、撤回、檔案中心操作等核心流程。

---

## 一、測試前準備

1. 測試帳號（至少 4 個）
- `A`（房主）
- `B`（普通成員）
- `C`（普通成員）
- `D`（普通成員）

2. 測試資源
- 圖片（jpg/png）
- 影片（mp4）
- 音頻（mp3）
- PDF
- 其他檔案（zip/txt）

3. 客戶端
- GoTradeTalk UI（至少 2 個瀏覽器視窗）
- Element（用於交叉比對 Matrix 事件結果）

---

## 二、群聊邀請/退出/移除複雜路徑

### Case G-01：複雜邀請路徑（你提出的主路徑）
1. `A` 建群，首次邀請 `B`。
2. `B` 加入後，`B` 主動退出群聊。
3. `A` 再次邀請 `B` 與 `C` 一起加入。
4. `C` 加入後主動退出。
5. `A` 邀請 `D` 加入。
6. `D` 加入後，`A` 再邀請 `C`。

驗收：
- 每次被邀請者都能在 UI 正常看到可加入狀態。
- 點擊加入後可即時進群，不需刷新。
- 不出現「房間版本 1 空房間占位」不可進問題。
- 退出者再次受邀時可正常加入。

### Case G-02：Allow Members to Invite 驗證
1. `A` 開啟「允許普通成員邀請」。
2. 由 `B` 邀請 `C`。

驗收：
- `C` 收到邀請並可成功加入。
- 關閉開關後，`B` 再邀請應被拒絕。

### Case G-03：房主移除成員
1. `A` 將 `B` 從群聊移除（kick）。

驗收：
- `B` 當前在聊天室畫面時，應自動退出到空佔位/不可輸入狀態。
- `B` 端收到「你已經被移出聊天室-xxx」通知。
- 房間內出現系統訊息（帶時間）顯示成員被移除。

## 三、附件草稿與送出流程

### Case F-01：草稿上傳與不送出
1. 在聊天室選擇檔案，等待上傳完成。
2. 不按送出，切換到其他聊天室，再切回。

驗收：
- 草稿附件仍存在於原聊天室草稿區。
- 不應在 timeline 生成正式訊息。

### Case F-02：取消草稿
1. 上傳草稿後，點 `✕` 取消。

驗收：
- 草稿 UI 消失。
- 伺服器端檔案被刪除（可透過下載連結不可用或日誌核對）。

### Case F-03：上傳失敗重試
1. 製造上傳失敗（可暫斷網/限流）。
2. 點單檔 `重試`。

驗收：
- 該檔案重新進入上傳流程並可成功。
- 不需重新選擇檔案。

### Case F-04：文字 + 多附件送出
1. 同時準備文字與多個已完成草稿附件。
2. 點送出。

驗收：
- 文字與附件都成功進入 timeline。
- 草稿區已送出附件被清空。

## 四、附件刪除/撤回與權限

### Case F-05：本人刪除（不限時間）
1. 用 `A` 發一個附件訊息。
2. 等待超過 5 分鐘後再刪除。

驗收：
- 仍可成功刪除（不限時間）。
- 房內出現系統訊息：`xxx撤回一個文件`。

### Case F-06：非本人刪除
1. `A` 發附件，`B` 嘗試刪除。

驗收：
- `B` 無法刪除 `A` 的附件。
- UI 提示權限不足（不可靜默失敗）。

---

## 五、檔案中心（中欄/右欄）

### Case L-01：中欄聊天室摘要
驗收：
- 僅顯示有附件的聊天室。
- 顯示 `聊天室名(附件數)` 與總容量。
- 排序由新到舊。

### Case L-02：右欄檔案列表
驗收：
- 顯示：勾選框、略縮圖/副檔名 icon、類型、副檔名、大小(MB)、操作。
- 操作含：預覽、下載、跳轉、刪除。

### Case L-03：搜尋與篩選
驗收：
- 中欄聊天室搜尋可用。
- 右欄檔名搜尋可用。
- 右欄類型篩選可用。

### Case L-04：批量刪除
驗收：
- 批量入口在右欄。
- 勾選框位於略縮圖前方。
- 批量刪除後，聊天區與檔案中心同步更新。
