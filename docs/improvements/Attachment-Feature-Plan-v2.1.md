# GoTradeTalk v2.1 附件與檔案管理開發計劃 (Attachment Feature Plan)

> **目標**：補齊聊天附件基礎能力（草稿上傳、可撤回文件消息、檔案中心、批量刪除）。
> **範圍**：`gotradetalk-ui` 前端 + `continuwuity`（Matrix homeserver）+ `hub-backend`（業務配置/策略）+ Matrix 事件同步。
> **原則**：每完成一個階段即回歸測試一次；附件撤回維持「僅本人可刪除，不限制時間」。

## Phase A: 附件草稿流與撤回規則（P0）
**目標**：先把資料正確性與安全規則打好，避免無效檔案堆積與越權刪除。

1. **附件流程改造（選擇即上傳，送出才發消息）**
   - [x] 檔案選擇後先上傳為「草稿附件」（不出現在聊天室 timeline）。
   - [x] 輸入框顯示待發送附件區（上傳中/成功/失敗/重試）。
   - [x] 點擊發送後，才建立正式文件消息（可與文字同送）。

2. **取消與清理**
   - [x] 待發送附件點擊「叉」時，立即刪除伺服器檔案。
   - [x] 補充未送出草稿附件清理（TTL 任務，24h）。

3. **文件撤回規則**
   - [x] 僅允許「上傳者本人」刪除/撤回該文件。
   - [x] 取消撤回時效限制（不限時間）。
   - [x] 撤回成功後寫入系統訊息：`xxx撤回一個文件`。
   - [x] 撤回後文件下載入口需失效，避免殘留可訪問連結。

4. **錯誤碼與提示**
   - [x] 配額超限、權限不足，統一可讀提示文案。
   - [ ] 前端依錯誤碼映射本地化提示（zh-CN/en 已完成，zh-TW 待補）。

### Phase A 驗收測試
- [ ] 測試 1：選擇附件後不送出，timeline 無文件消息。
- [ ] 測試 2：草稿附件按叉取消，伺服器檔案被刪除。
- [ ] 測試 3：歷史文件可撤回（不限時間），且非本人不可撤回。
- [ ] 測試 4：非上傳者嘗試刪除，應被拒絕。
- [ ] 測試 5：撤回後全員看到系統訊息，刷新後狀態一致。

## Phase B: 文件消息互動體驗（P0/P1）
**目標**：補齊聊天區可用性，行為接近 ChatGPT/Gemini。

1. **聊天區文件消息操作**
   - [x] 文件消息時間旁新增 `...` 操作入口。
   - [x] 目前功能：`刪除（撤回）`。
   - [x] 執行刪除後：文件消息消失 + 系統訊息顯示 + 實體檔案刪除。

2. **預覽與打開策略**
   - [x] 圖片：點擊放大/拖曳。
   - [x] 影片/音頻：可播放。
   - [x] PDF：可預覽。
   - [x] 其他類型：下載。

3. **失敗重試與可觀測性**
   - [x] 草稿附件上傳失敗可單檔重試。
   - [x] 前端事件追蹤：上傳、取消、送出、撤回、失敗原因。

### Phase B 驗收測試
- [ ] 測試 1：文字 + 多附件一次送出成功。
- [ ] 測試 2：文件消息 `... -> 刪除` 完整生效。
- [ ] 測試 3：撤回後其他端即時同步且刷新一致。
- [ ] 測試 4：圖片/影片/PDF/其他檔案行為符合預期。

## Phase C: 左側檔案中心與批量刪除（P1）
**目標**：提供帳號層級文件管理能力，按聊天室聚合與快速操作。

1. **檔案中心入口與列表**
   - [x] 左側功能欄新增資料夾圖示。
   - [x] 中欄改為聊天室摘要清單（僅顯示有附件的聊天室）。
   - [x] 中欄顯示：聊天室名稱、附件數量、該聊天室上傳總容量（可計算部分）。
   - [x] 中欄排序：由最新附件時間到最舊。

2. **單檔操作**
   - [x] 右欄顯示選中聊天室的附件列表。
   - [x] 右欄每筆顯示：略縮圖（或副檔名 icon）、檔案類型、副檔名、大小(MB)、操作。
   - [x] 每筆 `...`：`下載`、`跳到該聊天記錄點`、`刪除`。

3. **批量操作**
   - [x] 批量選擇入口移至右欄（檔案列表區）。
   - [x] 批量模式勾選框位於略縮圖前方。
   - [x] 批量模式可多選檔案，右欄頂部顯示刪除鍵。
   - [x] 批量刪除後同步更新聊天區與檔案中心。

4. **搜尋與篩選**
   - [x] 中欄支援聊天室名稱搜尋。
   - [x] 右欄檔名搜尋。
   - [x] 右欄類型篩選。

### Phase C 驗收測試
- [ ] 測試 1：分組與排序正確。
- [ ] 測試 2：單檔刪除、批量刪除皆只影響本人檔案。
- [ ] 測試 3：跳轉到聊天記錄定位正確。
- [ ] 測試 4：搜尋/篩選結果正確且性能可接受。
